# 按天拼图（Daily Collage）产品需求文档

## 1. 问题陈述

用户每天会用手机拍摄多张照片，但散落在相册中缺乏整理感。手动将同一天的照片拼成一张图耗时费力，市面拼图工具多为手动选图 + 手动排版，缺少「按日期自动分组、一键生成」的能力。用户需要一个能自动按日期整理照片并生成精美拼图的工具，同时保留灵活的手动微调空间。

## 2. 解决方案

提供一个 Android 原生应用，用户只需选择日期范围，应用自动从相册按天分组照片，智能选择最优布局，一键生成带日期水印的拼图并保存。同时支持丰富的手动编辑功能，让用户在自动化基础上精细调整每一张拼图。

## 3. 用户故事

1. 作为一个手机用户，我想选择一个日期范围，以便应用自动查找这段时间内的照片
2. 作为一个手机用户，我想看到照片按天自动分组，以便了解每天拍了哪些照片
3. 作为一个用户，我想应用根据图片数量和比例自动选择最优拼图布局，以便无需手动排版
4. 作为一个用户，我想从 13 种布局规则中手动选择（如竖排、横排、主图+副图、网格等），以便按个人喜好调整风格
5. 作为一个用户，我想选择输出比例（原始、1:1、4:3、3:4、16:9、9:16），以便适配不同社交平台的发布需求
6. 作为一个用户，我想在生成前看到低分辨率预览，以便确认效果再保存
7. 作为一个用户，我想点击预览图中的某个格子打开全屏裁切编辑器，以便精确调整该图片在格子内的可见区域
8. 作为一个用户，我想在裁切编辑器中看到完整原图（框外暗、框内亮），以便清楚知道裁掉了哪些内容
9. 作为一个用户，我想通过拖拽和缩放手势调整裁切位置，以便获得满意的构图
10. 作为一个用户，我想交换两张图片在拼图中的位置，以便优化视觉效果
11. 作为一个用户，我想通过左右箭头微调图片顺序，以便快速调整相邻图片位置
12. 作为一个用户，我想移除某一天中不想要的图片，以便只保留精选照片
13. 作为一个用户，我想为某一天的拼图添加额外图片，以便补充遗漏的照片
14. 作为一个用户，我想一键重置某天的全部编辑（删除、添加、裁切），以便恢复到自动选择状态
15. 作为一个用户，我想双击预览图全屏放大查看，以便检查细节
16. 作为一个用户，我想在全屏放大视图中也能通过点击交换图片，以便在更清晰的视图下操作
17. 作为一个用户，我想拼图自动添加日期水印，以便回看时知道拍摄日期
18. 作为一个用户，我想最终拼图以高分辨率（1080px）JPEG 格式保存到系统相册，以便随时分享
19. 作为一个用户，我想选择拼图后是否删除原图，以便节省手机存储空间
20. 作为一个用户，我想在删除原图时系统弹出确认对话框（Android 11+），以便防止误删
21. 作为一个用户，我想看到拼接进度（当前第几天/共几天），以便了解处理进度
22. 作为一个用户，我想拼图没有黑边白边（center-crop），以便获得干净的视觉效果
23. 作为一个用户，我想布局算法根据图片实际宽高比智能评分，以便减少不必要的裁切

## 4. 功能模块

### 4.1 图片查询与分组

- 通过 MediaStore 查询指定日期范围内的照片
- 按拍摄日期（`DATE_TAKEN`）分组，无拍摄时间时回退到 `DATE_ADDED`
- 每天的照片按时间正序排列

### 4.2 布局规划引擎

- **13 种布局规则**：
  - `AUTO`（智能推荐）、`VERTICAL`（竖排）、`HORIZONTAL`（横排）
  - `HERO_TOP`（顶部主图）、`HERO_LEFT`（左侧主图）、`HERO_RIGHT`（右侧主图）
  - `GRID_2/3/4`（等分网格）、`GRID_SQUARE`（正方形网格）
  - `FIT_2/3/4`（等比适配）
- **AUTO 模式算法**：生成多个候选布局，通过评分函数综合评估（总高度偏差 × 0.3 + 行高方差 × 0.4 + 裁切惩罚 × 0.3），选最优方案
- 支持自适应行排布，根据图片实际宽高比动态分配列宽

### 4.3 拼图渲染引擎

- 预览分辨率：360px 宽
- 输出分辨率：1080px 宽
- 采用 center-crop 缩放，消除黑边白边
- 支持裁切偏移（`CropOffset`：x/y 锚点 + 缩放倍率）
- 日期水印：右下角白字 + 半透明黑底圆角背景 + 文字阴影

### 4.4 输出比例适配

- 6 种输出比例，NATURAL 保持原始布局比例
- 其他比例通过 center-crop 适配目标比例

### 4.5 裁切编辑器

- 全屏 Dialog 展示完整原图
- 半透明黑色遮罩 + 白色裁切框标识可见区域
- 支持平移（单指拖拽）和缩放（双指捏合）手势
- 最小缩放限制确保图片始终覆盖裁切框
- 确认/重置/取消操作

### 4.6 图片编辑

- 交换任意两张图片位置
- 左右箭头逐步移动图片顺序
- 移除单张图片
- 从相册添加图片
- 一键重置某天的全部修改

### 4.7 图片保存与删除

- JPEG 90% 质量保存到 `Pictures/DailyCollage/` 目录
- 通过 MediaStore API 注册到系统相册
- 可选删除原图功能，Android 11+ 使用 `createDeleteRequest` 安全删除

## 5. 技术架构决策

| 决策项 | 选择 | 理由 |
|---|---|---|
| UI 框架 | Jetpack Compose + Material 3 | 现代声明式 UI，开发效率高 |
| 架构模式 | 单 Activity + ViewModel (MVVM) | 简单直接，无过度设计 |
| 状态管理 | MutableStateFlow / StateFlow | Kotlin 协程原生支持，与 Compose 无缝集成 |
| 图片加载 | 原生 BitmapFactory + Canvas | 避免第三方依赖，拼图场景需要精确控制 Bitmap |
| 缩略图解码 | inSampleSize 降采样 | 内存高效，预览用 160px，编辑器用 1200px |
| 权限适配 | 按 SDK 版本动态选择权限 | 兼容 Android 8.0 到最新版本 |
| 依赖注入 | 无（构造函数直接创建） | 应用规模小，无需 DI 框架 |
| 布局评分 | 多因子加权评分函数 | 平衡美观性和裁切量 |
| 坐标模型 | 归一化 0.0-1.0 坐标 | 分辨率无关，预览和输出可复用同一偏移值 |

## 6. 测试决策

### 测试原则

- 测试外部行为而非内部实现
- 优先覆盖核心域逻辑，UI 层依赖手动测试

### 建议测试模块

| 模块 | 测试重点 |
|---|---|
| CollageLayoutPlanner | 各布局规则的单元格分布正确性、AUTO 模式评分排序、边界情况（1张/20张图） |
| CropOffset | coerced() 边界值、CENTER 常量 |
| ImageGrouper | 日期分组正确性、空列表、跨时区 |
| OutputAspectRatioFitter | 各比例的裁切计算正确性 |
| GridCollageMaker | center-crop 坐标计算、水印位置 |

### 当前测试现状

项目目前无自动化测试，所有验证通过手动操作完成。

## 7. 范围外

- **云同步 / 账号体系**：纯本地应用，不涉及网络功能
- **视频拼接**：仅支持静态图片
- **跨设备同步**：不支持多设备间共享拼图配置
- **AI 智能选图**：不自动筛选"最佳"照片，由用户手动选择
- **自定义水印**：当前仅支持固定格式的日期水印
- **自定义间距/边框样式**：布局间距为固定值
- **iOS 版本**：仅 Android 平台

## 8. 补充说明

- **性能考虑**：预览使用 360px 低分辨率，最终输出才使用 1080px，避免编辑时内存压力
- **兼容性**：最低支持 Android 8.0（API 26），覆盖 99%+ 的活跃设备
- **存储路径**：拼图保存在系统 Pictures 目录的 `DailyCollage` 子文件夹下，方便用户在相册中查找
- **用户数据安全**：删除原图功能在 Android 11+ 通过系统级确认弹窗保护，低版本直接使用 ContentResolver.delete
